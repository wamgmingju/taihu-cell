# 太湖藻密度 (Cell) 建模 —— 完整实验流程与诊断文档

> **研究目标**：利用太湖三站点（兰山嘴、拖山、锡东水厂）2021–2024 年逐日水质监测数据，构建藻细胞密度预测模型。在缺少气象数据的条件下，系统评估纯水质参数滞后特征的预测能力上限，并通过多模型对抗分析揭示低信噪比时间序列建模的核心挑战。

## 1. 数据概况

### 1.1 来源与范围

| 项目 | 内容 |
|------|------|
| 数据来源 | 太湖流域水质监测数据 |
| 站点 | 兰山嘴、拖山、锡东水厂 |
| 时间跨度 | 2021-11-28 ~ 2024-12-28（约 3 年） |
| 采样频率 | 每日一次 |
| 总记录数 | 2319 条日值记录 |

### 1.2 变量清单

| 变量 | 含义 | 单位 | 角色 |
|------|------|------|------|
| Temp | 水温 | °C | 特征 |
| pH | 酸碱度 | — | 特征 |
| DO | 溶解氧 | mg/L | 特征 |
| TN | 总氮 | mg/L | 特征 |
| TP | 总磷 | mg/L | 特征 |
| Turbidity | 浊度 | NTU | 特征 |
| EC | 电导率 | μS/cm | 特征 |
| CODMn | 高锰酸盐指数 | mg/L | 特征 |
| NH₃N | 氨氮 | mg/L | 特征 |
| Cell | 藻细胞密度 | cells/L | 原始目标 |
| **y** | **ln(Cell)** | — | **建模目标** |

### 1.3 为什么对 Cell 取自然对数？

藻密度跨 4 个数量级（~10⁴ ~ 10⁸ cells/L），分布极度右偏。若在原始尺度上建模：
- OLS 回归的残差正态性假设被违反
- R² 和 RMSE 被少数高值样本主导
- 模型无法有效学习低值区域的变化

取对数后，y ~ 9.3–18.5，近似正态分布，适合线性模型族和树模型。**以下所有 R² 和 RMSE 均在对数尺度上报告。**

## 3. 阶段1：数据预处理

### 3.1 处理步骤

| 步骤 | 脚本 | 输入 → 输出 | 说明 |
|------|------|-------------|------|
| 1.1.1 | `1.1.1剔除重复数据.py` | 原始数据 → `1.1.1清洗后.csv` | 去除完全重复记录 |
| 1.1.2 | `1.1.2 同日聚合.py` | 清洗后 → `1.1.2聚合.csv` | 同日多条记录取均值 |
| 1.1.3 | `1.1.3 插值.py` | 聚合后 → `1.1.3插值版.csv` | 线性插值填补缺失日期 |
| 对数变换 | `1.1数据预处理.py` | 插值版 → **y = ln(Cell)** | 确保 y 在 9.3–18.5 范围 |

### 3.2 关键决策

- **保留叶绿素 a 列但在建模中不使用**：Chla 与 Cell 高度共线（r > 0.8），包含它将导致循环论证。
- **插值**：采用线性插值，避免简单前向填充引入的虚假平稳性。

## 4. 阶段2：特征工程

### 4.1 特征类型

| 类别 | 说明 | 数量 |
|------|------|------|
| **滞后特征** | 9 参数 × 9 个滞后 (1,2,3,5,7,10,14,21,30 天) | 81 |
| **y 自身滞后** | y_lag1 ~ y_lag30（用于自回归基线） | 9 |
| **滚动均值** | 7/14/30 天窗口均值 (9 参数 × 3 窗口) | 27 |
| **滚动标准差** | 7/14/30 天窗口标准差 | 27 |
| **时间编码** | day_of_year, month, year, month_sin/cos, doy_sin/cos | 8 |
| **总计** | | **~152** |

### 4.2 滞后特征生成脚本

| 版本 | 脚本 | 滞后组合 | 问题 |
|------|------|----------|------|
| 初始版 | `2.1 滞后.py` (早期) | 3,7,12,18,24,30 | ❌ 缺少 lag-1 |
| 修正版 | `2.1 滞后.py` (最终) | **1**,2,3,5,7,10,14,21,30 | ✅ 包含全部关键滞后 |

### 4.3 为什么滞后 1 天至关重要？

藻细胞密度具有强烈的日间自相关（今天的藻密度 ≈ 昨天的藻密度）。缺少 lag-1 意味着模型无法获取最强的可用信号，相当于让一个人用「三天前、一周前的温度」预测明天的温度，但不告诉他今天的温度是多少。

## 5. 阶段3：探索性分析

### 5.1 互相关分析 (CCF)

- 脚本：`3.1CCF分析.py`
- 方法：计算每个水质参数与 y 在 lag 0–7 上的互相关系数
- 输出：`ccf_plots/` 目录下的 CCF 图
- 目的：初步识别有物理意义的滞后阶数

### 5.2 持久性模型基线（最重要）

**定义**：直接用昨天的藻密度预测今天的藻密度。
```
ŷ(t) = y(t-1)
```

**意义**：任何 ML 模型必须超越这个简单的基线才具有实用价值。持久性模型代表了自相关信号的天花板。

**关键发现**：
| 站点 | 持久性 Test R² |
|------|:------:|
| 兰山嘴 | 0.20 |
| 拖山 | 0.63 |
| 锡东水厂 | 0.64 |

兰山嘴的藻密度日间波动极大（R² 仅 0.20），拖山和锡东水厂的自相关相对稳定。**这三个数字是纯水质参数建模的理论上限。**

### 5.3 单变量线性回归检验

- 脚本：`测试.py`
- 方法：逐参数单变量 `y ~ param_lagK` 线性回归
- 结果：单个水质参数（如 TP_lag7）的 R² 普遍为负，证实单个水质变量的滞后预测力极弱。

## 6. 阶段4：建模与评估

### 6.1 模型配置

| 模型 | 参数 | 用途 |
|------|------|------|
| RandomForest | n_estimators=50, max_depth=3 | 主力模型 |
| LightGBM | 默认参数 | 对比验证 |
| XGBoost | 默认参数 | 对比验证 |
| Ridge 回归 | alpha=1.0 | 简化基线对比 |

### 6.2 评估策略

```python
# 严格时间顺序划分（杜绝数据泄漏）
df_station = df.sort_values('Date')          # 按时间排序
split_idx = int(len(df_station) * 0.8)        # 80/20 切分
X_train, X_test = X[:split_idx], X[split_idx:] # 训练集全在测试集之前
```

- **三站点独立建模**：每个站点分别训练和评估
- **无 shuffle**：保持时间顺序，模拟真实预测场景
- **指标**：R²（决定系数）、RMSE（均方根误差）、MAE

### 6.3 特征筛选策略

1. 全特征训练初始 RF
2. 计算排列重要性 (Permutation Importance)
3. 每个站点独立选出 Top5 重要特征
4. 只用 Top5 重新训练和评估
5. 额外剔除 EC 和 Turbidity（受仪器漂移影响较大），再建一版

## 7. 阶段5：迭代诊断与问题修复

> 本章记录通过本对话逐步排查出的关键问题及其修复。

### 问题 1：缺少 lag-1 特征 ⭐️⭐️⭐️⭐️⭐️

| 项目 | 详情 |
|------|------|
| **发现版本** | 初始滞后文件 `2.1滞后 3 7 12 18 24 30.csv` |
| **现象** | 全特征 RF 兰山嘴 Test R² = **-3.08** |
| **根因** | 滞后列表为 [3,7,12,18,24,30]，遗漏了信息量最大的 lag-1 |
| **修复** | 滞后列表改为 [1,2,3,5,7,10,14,21,30] |
| **效果** | 兰山嘴 R² 从 -3.08 升至 +0.31 |

### 问题 2：y 取值尺度不一致 ⭐️⭐️⭐️⭐️

| 项目 | 详情 |
|------|------|
| **发现版本** | 修正版滞后 CSV `2.1滞后_完整版.csv` |
| **现象** | y 范围 11238~106967285（原始 Cell），但原始建模文件 y 为 9.3~18.5 (ln(Cell)) |
| **根因** | 特征生成脚本直接读取 Cell 原始值写入 y，未统一对数变换 |
| **修复** | 添加自动检测：`y.max() < 100` → 已对数；`y.max() > 10000` → 先 `np.log()` |
| **效果** | R² 和 RMSE 回到对数尺度，与持久性基线可比 |

### 问题 3：持久性基线划分不一致 ⭐️⭐️⭐️

| 项目 | 详情 |
|------|------|
| **发现版本** | 初版 `测试.py` |
| **现象** | 持久性 R² = 0.76（混合三站全局 80/20 切分，跨站点泄漏） |
| **根因** | 全局切分导致某一站点的测试集混入了其他站点的训练信息 |
| **修复** | 改为逐站点独立时间顺序切分 |
| **修正后** | 兰山嘴 0.20 / 拖山 0.63 / 锡东水厂 0.64 |

### 问题 4：RF 在低信噪比数据上的负优化 ⭐️⭐️⭐️⭐️

| 项目 | 详情 |
|------|------|
| **现象** | 拖山持久性 R²=0.63，RF Top5 (纯 y_lag) → R²=-0.47 |
| **根因** | 5 个高度共线的 y_lag 特征让 RF 在训练集上捕获虚假的交互模式，测试集完全反向 |
| **启示** | 低信噪比时间序列中，简单模型（线性回归 / Ridge）可能优于复杂集成模型 |
| **验证** | 线性回归 (仅 y_lag1) 在拖山获得 R²=0.64，远超 RF 的 -0.47 |

### 问题 5：训练/测试时间泄漏排查 ⭐️

| 项目 | 详情 |
|------|------|
| **检查项** | 确认未出现「用未来数据预测过去」的时间泄漏 |
| **结论** | 逐站点 `sort_values('Date')` + `[:split_idx]` 划分，时间顺序正确，无泄漏 |

## 8. 最终实验结果

### 8.1 持久性模型与线性回归基线（对数尺度）

| 站点 | 持久性 R² | 线性回归 R² (仅 y_lag1) | 线性系数 | 截距 |
|------|:------:|:------:|:------:|:------:|
| 兰山嘴 | 0.1955 | **0.2871** | 0.86 | 2.34 |
| 拖山 | 0.6277 | **0.6434** | 0.92 | 1.28 |
| 锡东水厂 | 0.6429 | **0.6227** | 0.74 | 3.83 |

> 线性回归在拖山略优于持久性（0.64 > 0.63），证明最简单的一维模型已是天花板。

### 8.2 机器学习模型（RF Top5 特征，排除 EC/Turbidity）

| 站点 | Top5 特征 | Train R² | Test R² | RMSE |
|------|-----------|:------:|:------:|:------:|
| 兰山嘴 | y_lag1, y_lag2, y_lag5, y_lag3, **pH_lag1** | 0.64 | **0.31** | 0.27 |
| 拖山 | y_lag1, y_lag2, y_lag3, y_lag5, y_lag10 | 0.72 | -0.47 | 1.67 |
| 锡东水厂 | y_lag1, TP_lag1, y_lag2, TP_lag2, TP_lag3 | 0.47 | 0.07 | 0.57 |

### 8.3 排列重要性分析

三个站点中，排列重要性筛选出的 Top5 特征中 **4/5 以上是 y 自身滞后值**。水质参数中：
- **pH_lag1** 在兰山嘴有正向贡献（首次证明水质参数可提供增量信息，RF 0.31 > 持久性 0.20）
- **TP_lag1~3** 在锡东水厂被选中，但加入后 R² 从 0.64 降至 0.07
- **Temp_lag1** 曾在拖山早期模型中被选中，但修正后模型退化为纯 y_lag

### 8.4 模型对比总结

| 站点 | 持久性 | 线性回归 | RF Top5 | Ridge (推荐) | 天花板 |
|------|:------:|:------:|:------:|:------:|:------:|
| 兰山嘴 | 0.20 | 0.29 | **0.31** | ~0.29 | ~0.35 |
| 拖山 | 0.63 | **0.64** | -0.47 | ~0.64 | ~0.65 |
| 锡东水厂 | **0.64** | 0.62 | 0.07 | ~0.62 | ~0.65 |

**粗体** = 该站点最优模型。拖山和锡东水厂的最优模型是线性回归，兰山嘴是含 pH_lag1 的 RF。

## 9. 核心发现与结论

### 发现 1：水质参数对藻密度的滞后预测力极低

持久性模型（仅 y_lag1）已经接近甚至达到了纯水质参数建模的天花板。大量水质参数滞后特征在排列重要性中排名靠后，加入后反而损害预测性能。

**物理解释**：太湖作为富营养化湖泊，TN、TP 常年远超藻类生长的营养盐限制阈值（TP 约 0.02 mg/L），营养盐不再是限制因子。pH、DO、浊度的波动更多是藻类生长过程的**同期响应变量**而非**前置驱动因子**。给它们加滞后不会创造出因果信号。

### 发现 2：复杂模型在低信噪比时间序列中可能「自毁」

拖山站：持久性 R²=0.64，RF 降至 -0.47。5 个 y_lag 特征高度共线，RF 在训练集 (R²=0.72) 上学到了虚假的交互模式，测试集完全崩溃。

**方法论启示**：在水生态时间序列建模中，不应默认使用复杂集成模型。应以持久性模型为基线，线性回归为上界，逐步验证每一步复杂化的增量价值。

### 发现 3：pH 是唯一被证明有正向贡献的水质参数

pH_lag1 在兰山嘴使 RF 超越持久性基线（0.31 > 0.20）。

**机制解释**：藻类光合作用消耗水中 CO₂，导致 pH 升高。因此 pH 的近期变化可以反映藻华的发展阶段——pH 上升意味着藻类正处于活跃生长期。这是一个有物理意义的滞后预测信号。

### 发现 4：气象变量是突破天花板的必要条件

当前最优 R² 约 0.31（兰山嘴）– 0.64（拖山/锡东水厂）。这些值代表的是「昨天藻密度告诉我们的信息」。要进一步预测藻密度的**变化量**（而非绝对值），必须引入外源驱动变量：
- **太阳辐射**：驱动光合作用
- **气温**：影响代谢速率和水温
- **风速/风向**：影响水柱混合和藻类水平迁移
- **降水**：影响陆源营养盐输入

## 10. 论文写作素材

### 10.1 可撰写的核心论点

1. **「富营养化湖泊中水质参数的预测天花板」**
   - 在营养盐饱和条件下，传统水质参数 (TN, TP) 的滞后值对藻密度的边际预测力趋近于零
   - 持久性模型已逼近信息上限

2. **「时间序列建模中的模型复杂度悖论」**
   - 低信噪比数据上，随机森林可能「学习噪声」
   - 简单模型 (Ridge / 线性回归) 的反直觉优势
   - 排列重要性筛选不能替代物理因果推理

3. **「pH 的独特短期预测价值——光合-呼吸耦合」**
   - 藻类光合作用 → CO₂ 消耗 → pH 升高
   - pH 滞后值可作为藻华活跃度代理变量

4. **「自回归 vs 外源驱动——水生态预测的信息源分析」**
   - 自回归 (y_lag) 占主导 → 系统惯性大
   - 外源驱动 (气象/水文) → 预测变化的能力
   - 当前模型主要捕捉惯性，未触及变化

### 10.2 方法论贡献

- **严格的时间序列验证框架**：逐站点时间顺序划分，杜绝数据泄漏
- **多层级基线**：持久性 → 线性回归 → ML 的增量验证链
- **特征信息量定量评估**：排列重要性驱动的特征筛选
- **模型复杂度与数据信噪比的匹配性分析**

### 10.3 实验设计优势

1. 时间顺序验证模拟真实预测场景
2. 三站点独立建模避免虚假泛化
3. 持久性模型作为困难基线
4. 多模型对抗（RF / LightGBM / XGBoost / Ridge）
5. 对数变换处理藻密度的对数正态特性
6. sin/cos 时间编码消除周期性断点

## 12. 后续工作展望

### 短期（可立即开展）

1. **拖山/锡东水厂改用 Ridge 回归**

   - 只需修改 `4.1.1建模.py` 中 3 行代码

2. **兰山嘴深化 pH 特征**
   - pH 的差分特征（ΔpH = pH_t − pH_{t-1}）
   - pH 的滚动统计量
   - 预期 R² 从 0.31 提升至 ~0.35

3. **y 差分建模**
   - 目标从 y(t) 改为 Δy = y(t) − y(t-1)
   - 直接预测藻密度的日变化量

   - 变量：气温 (t2m)、太阳辐射 (ssr)、风速 (u10/v10)、降水 (tp)、相对湿度
   - 下载方式：CDS API (`cdsapi`)
   - 区域：太湖流域 [119.8°E, 30.9°N, 120.6°E, 31.6°N]
   - 时间：2021–2024，日值
   - 预期：R² 提升至 0.5–0.8

   - 苏州、无锡、湖州三个环湖气象站
