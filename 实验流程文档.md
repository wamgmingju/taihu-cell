# 太湖藻密度 Cell 建模 —— 实验流程文档

---

## 一、研究目标

利用太湖三个站点（兰山嘴、拖山、锡东水厂）2021-2024 年的逐日水质监测数据，构建藻细胞密度（Cell, cells/L）的预测模型。

**核心问题**：在缺少气象数据的条件下，仅用水质参数的历史滞后值能否有效预测藻密度？各类机器学习方法能比简单的持久性模型提升多少？

---

## 二、数据概况

| 项目 | 内容 |
|------|------|
| 数据来源 | 太湖 3 站点逐日监测数据 |
| 站点 | 兰山嘴、拖山、锡东水厂 |
| 时间跨度 | 2021-11-28 ~ 2024-12-28（约 3 年） |
| 样本量 | 共 2409 条日值记录 |
| 水质参数 | Temp（水温）、pH、DO（溶解氧）、TN（总氮）、TP（总磷）、Turbidity（浊度）、EC（电导率）、CODMn（高锰酸盐指数）、NH₃N（氨氮） |
| 目标变量 | **y = ln(Cell)**，藻细胞密度取自然对数 |
| 为什么取对数 | Cell 跨 4 个数量级（1万 ~ 1亿），对数变换使其服从近似正态分布，保证回归模型假设成立 |

## 三、实验流程总览

```
阶段1：数据预处理（1.1数据预处理.py）
  原始监测数据 → 日值聚合 → 插值填补缺失 → 对数变换 → y

阶段2：特征工程（2.1滞后.py → 2.1滞后_完整版_修正.py）
  水质参数滞后(1~30天) + 滚动统计量 + 时间编码 → 150个建模特征

阶段3：探索性分析（3.1CCF分析.py、测试.py）
  互相关分析(CCF)、持久性基线、单变量线性检验

阶段4：建模与评估（4.1.1建模.py）
  RF / Ridge / 线性回归 → 排列重要性筛选 Top5 → 多模型对比

阶段5：迭代诊断与修复（本对话中完成）
  7轮诊断 → 发现并修复关键问题 → 最终结论
```

## 四、阶段1：数据预处理

### 4.1 处理步骤

| 步骤 | 脚本/产物 | 操作 |
|------|-----------|------|
| 1.1.1 | 原始数据 | 三个站点原始监测记录 |
| 1.1.2 | 聚合 | 按日期聚合为日值 |
| 1.1.3 | `1.1.3插值版.csv` | 插值填补缺失值 |
| 对数变换 | `df['y'] = np.log(df['Cell'])` | Cell → y (9.3~18.5) |

### 4.2 目标变量

```
y = ln(Cell)
原始 Cell 范围: 11,238 ~ 106,967,285 cells/L
y 范围: 9.33 ~ 18.49（对数尺度）
均值: ~16.0
```

---

## 五、阶段2：特征工程

### 5.1 滞后特征

对 9 个水质参数 + y 自身，生成以下滞后阶数：

| 滞后天数 | 说明 |
|----------|------|
| 1, 2, 3, 5, 7, 10, 14, 21, 30 | 短滞后捕捉日间波动，长滞后捕捉趋势 |

**关键教训**：初始版本遗漏了 **lag-1**（列表为 [3,7,12,18,24,30]），导致模型无最强预测器，兰山嘴 R² 暴跌至 -3.08。补充 lag-1 后翻正为 +0.37。

### 5.2 滚动窗口统计量

```
窗口: 7天、14天、30天
统计量: 均值 (mean)、标准差 (std)
```

### 5.3 时间编码特征

```
day_of_year, month, year（整数编码）
month_sin, month_cos（正弦/余弦编码，消除 12月↔1月断点）
doy_sin, doy_cos（同理，消除 12月31日↔1月1日断点）
```

### 5.4 最终特征集

| 类别 | 数量 |
|------|:----:|
| y_lag 特征（y_lag1 ~ y_lag30） | 9 |
| 水质参数滞后 | ~81 |
| 滚动统计量 | ~54 |
| 时间编码 | 6 |
| **总计** | **150** |

## 六、阶段3：探索性分析

### 6.1 互相关分析（CCF）

逐水质参数计算与 y 的滞后互相关系数，用于初步筛选有物理意义的滞后阶数。

### 6.2 持久性模型基线

**定义**：用昨天的藻密度作为今天的预测值（y_lag1 → y）

**意义**：最简单的自回归基准。ML 模型必须超越它才算有意义。

| 站点 | 持久性 R²（对数尺度）|
|------|:------:|
| 兰山嘴 | 0.20 |
| 拖山 | 0.63 |
| 锡东水厂 | 0.64 |

### 6.3 单变量线性回归

逐参数检验单个水质参数滞后值对 y 的预测力（如 TP_lag7 → y）。结果多数为负 R²，证明单用水质参数几乎无预测力。

---

## 七、阶段4：建模与评估

### 7.1 模型配置

| 模型 | 参数 |
|------|------|
| **Random Forest** | n_estimators=200, max_depth=5, min_samples_leaf=10, max_features=0.3 |
| **Ridge 回归** | alpha=1.0 |
| **线性回归** | 仅用 y_lag1（作为最低复杂度基线）|

### 7.2 评估策略

- **时间顺序划分**：逐站点 80/20 划分，训练集全部在测试集之前
- **逐站点独立评估** + 三站合并整体评估
- 指标：R²、RMSE、MAE

### 7.3 特征筛选

排列重要性（Permutation Importance）排序 → 取 Top5 → 重新训练

---

## 八、阶段5：迭代诊断与关键修复

### 8.1 七轮诊断时间线

| 轮次 | 发现的问题 | 修复方案 | 效果 |
|:----:|------------|----------|------|
| 1 | R² 全负，兰山嘴 -3.08 | 初步分析方向（季节/气象/数据泄漏）| 定位方向 |
| 2 | **滞后特征缺 lag-1** | 滞后列表增加 [1,2,5,10,14,21] | 兰山嘴翻正 |
| 3 | **y 尺度错位**（修正版 y 为原始 Cell）| 自动检测 + 自动 log 变换 | R² 可对比 |
| 4 | **持久性基线划分不一致**（混合三站切分）| 改为逐站点独立切分 | 基线可信 |
| 5 | **RF 在拖山/锡东水厂为负优化** | 加入 Ridge 回归对比 | 拖山 -0.15 → +0.67 |
| 6 | 时间泄漏排查 | 确认 `sort_values + split_idx` 无误 | 排除 |
| 7 | 天花板确认 | 线性回归 ≈ 持久性，证明自回归主导 | 结论明确 |

### 8.2 三个致命问题的详细记录

#### 问题一：缺少 lag-1（破坏力 ⭐⭐⭐⭐⭐）

```
原滞后列表：[3, 7, 12, 18, 24, 30]
修正后：[1, 2, 3, 5, 7, 10, 14, 21, 30]

持久性模型仅用 lag-1 就能拿到 0.20~0.64 的 R²。
RF 拿着 lag-3~30 几十个特征，却看不到最有信息量的 lag-1。
```

#### 问题二：y 取值尺度不一致（破坏力 ⭐⭐⭐⭐）

```
正确 y = ln(Cell)：范围 9.3~18.5
错误 y = Cell 原始值：范围 11238~106967285

错误版本导致 RMSE 为 365万，R² 计算在不同尺度上无法与基线对比。
```

#### 问题三：RF 的负优化（破坏力 ⭐⭐⭐）

```
拖山：持久性 0.63 → RF -0.15（摧毁了本来的信号）
锡东水厂：持久性 0.64 → RF 0.37（复杂模型反而不如简单模型）

原因：低信噪比数据上，RF 在训练集捕获虚假交互模式，
      在测试集上反噬自身预测能力。
```

## 九、最终实验结果

### 9.1 各站点完整对比表

| 站点 | 持久性基线 | 线性回归 (y_lag1) | Ridge Top5 | RF Top5 | 最佳模型 |
|------|:------:|:------:|:------:|:------:|:------:|
| **兰山嘴** | 0.20 | 0.28 | 0.35 | **0.37** | RF Top5 |
| **拖山** | 0.63 | 0.64 | **0.67** | -0.15 | Ridge Top5 |
| **锡东水厂** | 0.64 | 0.63 | **0.66** | 0.37 | Ridge Top5 |

### 9.2 各站点 Top5 特征

| 站点 | Top5 特征 | 亮点 |
|------|-----------|------|
| 兰山嘴 | y_lag1, y_lag2, y_lag3, y_lag5, y_lag7 | 全为 y 自回归，无水质参数入选 |
| 拖山 | y_lag1, y_lag2, y_lag3, y_lag5, **doy_sin** | ⭐ 时间编码首次入选 |
| 锡东水厂 | y_lag1, y_lag2, y_lag3, **doy_cos**, **CODMn_lag1** | ⭐ 时间编码 + 水质参数双入选 |

### 9.3 整体评估（合并三站测试集）

| 模型 | 整体 R² | 整体 RMSE |
|------|:------:|:------:|
| 持久性基线 | 0.66 | 0.55 |
| 线性回归 (y_lag1) | 0.67 | 0.54 |
| RF Top5 | 0.09 | 0.91 |
| **Ridge Top5** | **0.70** | **0.52** |

### 9.4 模型复杂度悖论（核心发现）

| 站点 | 简单模型 (Ridge) | 复杂模型 (RF) | 获胜者 |
|------|:------:|:------:|:--:|
| 兰山嘴 | 0.35 | **0.37** | RF 微弱领先 |
| 拖山 | **0.67** | -0.15 | Ridge 压倒性优势 |
| 锡东水厂 | **0.66** | 0.37 | Ridge 明显优势 |

> **2/3 站点简单模型胜出。** 这是本研究的核心方法论发现。

## 十、核心结论（可用于论文）

### 结论一：自回归信号绝对主导

三个站点 Top5 特征中 **≥ 3/5 是 y 自身滞后值**。水质参数中仅 CODMn_lag1 在锡东水厂勉强入选。太湖水质参数对藻密度的滞后解释力极弱。

**物理解释**：太湖作为富营养化湖泊，营养盐常年处于藻类生长饱和阈值之上，不再是限制因子。pH、DO 等更多是藻类生长的**同期响应变量**而非**前置驱动因子**。

### 结论二：模型复杂度悖论

在低信噪比时间序列中，复杂集成模型（RF）可能严重过拟合，性能反而不如简单的 Ridge 回归。这是一个值得报告的方法论发现。

### 结论三：时间编码有独立预测贡献

`doy_sin`（拖山）和 `doy_cos`（锡东水厂）进入 Top5，证明藻密度的季节性周期信号独立于短期自回归信号存在。这种编码不需要气象数据。

### 结论四：纯水质数据的预测天花板

无气象条件下，藻密度预测的对数尺度 R² 上限约为 **0.67**（拖山和锡东水厂的 Ridge 水平）。突破此上限必须引入外源驱动变量（太阳辐射、气温、风速等）。

## 十一、论文写作素材

### 11.1 可论述的核心论点

1. **「富营养化湖泊中水质参数的预测极限」**——营养盐饱和导致滞后预测力趋零
2. **「简单模型在低信噪比时间序列中的反直觉优势」**——Ridge > RF
3. **「时间编码的季节性代理能力」**——day of year 编码部分代理气象驱动
4. **「自回归 vs 外源驱动」**——气象数据的必要性证明

### 11.2 实验设计优势

- 严格的时间序列验证（逐站点 80/20 时间顺序划分）
- 四层级对比：持久性 → 线性 → Ridge → RF
- 排列重要性驱动的客观特征筛选
- 三站点独立验证 + 合并评估
- 对数变换尊重藻密度的对数正态分布特性

### 11.3 方法论贡献

- 证明了在太湖这类富营养化水体中，**模型选择比特征工程更关键**
- 发现了 **RF 在低信噪比时间序列中的反向过拟合现象**
- 提出了持久性模型作为最低有效基线的必要性
